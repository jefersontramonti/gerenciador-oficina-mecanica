## ğŸ¯ Key Takeaways

1. **Access Token in Memory**

   - Automatically restored via refresh token
   - XSS cannot steal it

2. **Refresh Token in HttpOnly Cookie**
   - Persists across refreshes
   - Browser manages it automatically
   - XSS cannot read it

   - Auto-refresh on token expiry
   - User doesn't notice token management

   - Assume XSS can read everything in localStorage
   - Tokens NEVER touch localStorage
